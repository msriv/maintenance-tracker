Release a new version of MotoTracker. Argument: version tag (e.g. v0.0.5-dev). If no argument is provided, look at the latest GitHub release tag and increment the patch version.

Follow these steps exactly, stopping if any step fails:

## Step 1 — Determine version

If `$ARGUMENTS` is provided, use it as the version tag. Otherwise:
- Run `gh release list --limit 1` to get the latest tag
- Increment the numeric patch portion (e.g. v0.0.4-dev → v0.0.5-dev)
- Confirm the chosen tag with the user before proceeding

## Step 2 — Local build

Generate a debug keystore if it doesn't exist:
```
mkdir -p $HOME/.android
keytool -genkeypair -v \
  -keystore $HOME/.android/debug.keystore \
  -storepass android -alias androiddebugkey -keypass android \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -dname "CN=Android Debug,O=Android,C=US" 2>/dev/null || true
```

Build the release APK:
```
./gradlew assembleRelease --no-daemon \
  -Pandroid.injected.signing.store.file=$HOME/.android/debug.keystore \
  -Pandroid.injected.signing.store.password=android \
  -Pandroid.injected.signing.key.alias=androiddebugkey \
  -Pandroid.injected.signing.key.password=android
```

If the build fails, print the errors clearly and STOP. Do not proceed to commit or release.

If the build succeeds, locate and rename the APK:
```
VERSION=<tag without leading v>
SRC=$(find app/build/outputs/apk/release -name "*.apk" | head -1)
DEST="app/build/outputs/apk/release/MotoTracker-${VERSION}.apk"
mv "$SRC" "$DEST"
```

## Step 3 — Commit and push

Stage all modified tracked files and any untracked files that belong to the project (exclude build outputs, .apk files, and anything in .gitignore).

Write a concise commit message summarising the changes using `git diff --stat HEAD` and `git status` to understand what changed.

Push to main:
```
git push origin main
```

## Step 4 — Create GitHub release with APK

Check if the tag already exists on GitHub:
```
gh release view <tag> 2>/dev/null
```

If it exists, delete and recreate it (the build may have changed):
```
gh release delete <tag> --yes 2>/dev/null || true
git tag -d <tag> 2>/dev/null || true
git push origin :refs/tags/<tag> 2>/dev/null || true
```

Create the release and attach the APK in a single command:
```
gh release create <tag> \
  --title "MotoTracker <version>" \
  --notes "<auto-generated release notes based on commits since last tag>" \
  --prerelease \
  "app/build/outputs/apk/release/MotoTracker-<version>.apk"
```

For release notes, run `git log <previous-tag>..HEAD --oneline` to generate a bullet list of changes.

## Step 5 — Confirm

Print the release URL and confirm the APK is attached by running:
```
gh release view <tag>
```
